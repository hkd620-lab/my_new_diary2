# AI EDIT GUIDE (HARU / my_new_diary2)
# 목적: AI가 수정을 하면서 개념을 잊거나, 멀쩡한 코드를 되돌리는 사고를 방지한다.

---

## 0. 이 문서는 무엇인가
이 문서는 **AI(ChatGPT, Claude 포함)를 제어하기 위한 작업 지침서**다.  
코드보다 우선한다.  
모든 수정 요청 시 이 지침을 기준으로 한다.

---

## 1. 프로젝트 절대 전제 (변경 불가)

### 1-1. Write 화면 개념 (가장 중요)
- 일기는 **선택 → 입력 → 저장** 구조다.
- 모든 섹션은 **처음에는 선택만 한다**.
- 선택된 섹션만 입력 UI가 나타난다.
- 선택되지 않은 섹션은 화면에 보이지 않는다.

❌ 잘못된 예:
- Write 화면에서 모든 섹션 입력창을 한 번에 보여줌

✅ 올바른 예:
- [보람 있었던 일] 선택 → 보람 입력창만 등장

---

## 2. 파일 역할 고정 (AI는 반드시 준수)

### 2-1. Write 관련
- **src/pages/Write.tsx**
  - 유일한 작성 화면
  - 화면 조립 역할
  - 로직 최소화

### 2-2. 컴포넌트
- src/components/AchievementInput.tsx  
  → 보람 있었던 일 입력 전용
- src/components/TextSection.tsx  
  → 공용 텍스트 섹션 입력
- (추가 시 역할 1개만 담당)

---

## 3. 수정 요청 규칙 (AI에게 그대로 전달)

### 3-1. 파일 명시 규칙 (필수)
❌ "Write 화면 수정해줘"  
✅ "src/pages/Write.tsx 파일의 섹션 선택 로직만 수정해줘"

### 3-2. 출력 규칙
- 수정 대상 파일 **1개만**
- 반드시 `cat` 전체 코드로 출력
- 부분 코드, diff, 설명 금지

## 7. 작업 원칙 요약
- 작은 수정만 한다
- 한 파일만 고친다
- 개념을 먼저 확인하고 코드를 고친다
- 헷갈리면 즉시 중단한다

이 지침은 프로젝트가 끝날 때까지 유지한다.
