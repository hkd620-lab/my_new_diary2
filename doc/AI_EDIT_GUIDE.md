# AI EDIT GUIDE (HARU / my_new_diary2)

## 목적
AI 수정 시 개념 오류 및 코드 되돌림 방지.

---

## 0. 이 문서의 역할
- AI(ChatGPT, Claude 포함) 제어 작업 지침서.
- 코드보다 우선.
- 모든 수정 요청 시 지침 준수.

---

## 1. 프로젝트 절대 전제

### 1-1. Write 화면 개념
- 일기: 선택 → 입력 → 저장 구조.
- 모든 섹션: 처음엔 선택만.
- 선택된 섹션만 입력 UI 표시.
- 선택되지 않은 섹션은 숨김.

❌ 잘못된 예:
- Write 화면에 모든 섹션 입력창 동시 표시.

✅ 올바른 예:
- [보람 있었던 일] 선택 → 보람 입력창만 표시.

---

## 2. 파일 역할 고정

### 2-1. Write 관련
- **src/pages/Write.tsx**
- 유일한 작성 화면.
- 화면 조립 및 최소 로직 담당.

### 2-2. 컴포넌트
- src/components/AchievementInput.tㅁsx
- 보람 있었던 일 입력 전용.
- src/components/TextSection.tsx
- 공용 텍스트 섹션 입력.
- (추가 시 역할 1개만 담당)

---

## 3. 수정 요청 규칙

### 3-1. 파일 명시 규칙
❌ "Write 화면 수정해줘"
✅ "src/pages/Write.tsx 파일의 섹션 선택 로직만 수정해줘"

### 3-2. 출력 규칙
- 수정 대상 파일 1개만.
- `cat` 전체 코드로 출력.
- 부분 코드, diff, 설명 금지.

---

## 4. 절대 수정 금지 파일
AI는 아래 파일을 수정/제안 금지.

- src/App.tsx
- src/main.tsx
- src/layouts/AppLayout.tsx
- 라우터 구조 전체

## 5. 작업 원칙 요약
- 작은 수정만.
- 한 파일만 수정.
- 개념 확인 후 코드 수정.
- 헷갈리면 중단.

이 지침은 프로젝트 종료까지 유지.
